\begin{abstract}
Breast cancer is one of the leading causes of death worldwide, so detecting it early and accurately is very important. In this study, we used the UCI Wisconsin Breast Cancer dataset to compare how well different machine learning models can classify tumors as either malignant or benign.

I first tested five basic models: Logistic Regression, Support Vector Machine (SVM), K-Nearest Neighbors (KNN), Decision Tree, and Naive Bayes. After that, I improved the models using techniques such as hyperparameter tuning (GridSearchCV), ensemble methods like Random Forest and Gradient Boosting, and a Soft Voting Ensemble that combines the best models.

The number of features was also reduced from 30 to 15 using SelectKBest to make the model simpler and more efficient. The tuned SVM gave the best individual performance with an F1 score of 0.9790 and ROC-AUC of 0.9931. The Voting Ensemble performed even better overall, achieving an ROC-AUC of 0.9970.

These results show that improving and combining simple machine learning models can give very high accuracy, without needing complex deep learning methods.
\end{abstract}

\section{Introduction}

Breast cancer is one of the most common and life-threatening diseases affecting millions of people around the world. According to the World Health Organization, it accounts for about 12\% of all new cancer cases each year~\cite{who2023}. Because of this, early and accurate diagnosis is extremely important, as it can greatly improve treatment success and increase survival rates.

In recent years, machine learning (ML) has become a valuable tool to support medical professionals in diagnosing breast cancer using clinical data. Compared to more complex deep learning approaches, classical ML methods are easier to interpret and require less computational power. This makes them especially useful in healthcare settings, where understanding how a model makes decisions is just as important as its accuracy.

In this study, the publicly available UCI Wisconsin Breast Cancer Diagnostic (WDBC) dataset~\cite{uci_wdbc} was used. This dataset contains 569 samples, each with 30 numerical features extracted from digitized images of fine needle aspirate (FNA) of breast masses. Each sample is labeled as either malignant (212 cases) or benign (357 cases). Since the dataset is slightly imbalanced, stratified splitting was applied to ensure fair representation of both classes during training and testing.

The main contributions of this paper are as follows:
\begin{itemize}
\item A structured evaluation of five classical machine learning models as baseline approaches on the WDBC dataset.
\item Performance improvements using techniques such as hyperparameter tuning, ensemble learning, and feature selection with SelectKBest.
\item An analysis of model robustness by testing how the best-performing model behaves under different levels of Gaussian noise.
\item Identification of the most important features for breast cancer classification using Random Forest feature importance.
\end{itemize}

The rest of the paper is organized as follows. Section II reviews related work. Section III explains the methodology. Section IV presents the results. Section V discusses the findings, and Section VI concludes the study.

\section{Related Work}

Several studies have explored the use of classical machine learning methods for breast cancer diagnosis using the UCI WDBC dataset. The dataset itself was first introduced by Wolberg et al.~\cite{wolberg1995}, who demonstrated how machine learning techniques could support the cytological diagnosis of breast tumors based on FNA measurements.

Later, Akay~\cite{akay2009} showed that using Support Vector Machines (SVM) together with feature selection can achieve very high accuracy (around 99.51\%). This work highlighted an important idea: removing unnecessary or irrelevant features can significantly improve model performance, especially when working with high-dimensional medical data.

Olson et al.~\cite{olson2017}, through the PMLB benchmark study, evaluated many different machine learning models across several datasets, including those from the UCI repository. Their results showed that ensemble methods such as Random Forest and Gradient Boosting often perform better than individual models, particularly in medical classification problems.

Similarly, Exarchos et al.~\cite{exarchos2012} found that combining multiple models through ensemble techniques leads to more reliable and accurate predictions compared to using a single model. Supporting this, Breiman~\cite{breiman2001} introduced Random Forests and demonstrated how they reduce overfitting and improve stability compared to individual decision trees.

While these studies mainly focus on improving prediction accuracy, this paper takes a broader approach. In addition to achieving strong performance, I also examine how models can be systematically improved, identify the most important features influencing predictions, and test how robust the models are when noise is introduced into the data. This provides a more complete evaluation, which is important for real-world clinical applications.

\section{Methodology}

\subsection{Dataset}

The UCI Wisconsin Breast Cancer Diagnostic dataset~\cite{uci_wdbc} used in this study contains 569 patient samples, each described by 30 numerical features. These features are extracted from digitized images of fine needle aspirate (FNA) of breast masses and represent different characteristics of cell nuclei, such as radius, texture, perimeter, area, smoothness, compactness, concavity, symmetry, and fractal dimension.

For each of these characteristics, three types of values are recorded: the mean, the standard error, and the worst (maximum) value observed in the image. This provides a detailed representation of the cell structure for each sample.

Each instance in the dataset is labeled as either malignant (Class 0, 212 samples) or benign (Class 1, 357 samples). The dataset is clean and does not contain any missing values, making it well-suited for machine learning analysis.

\subsection{Preprocessing}

Before training the models, the dataset was carefully prepared through a series of preprocessing steps to ensure reliable and unbiased results.

\begin{enumerate}
\item \textbf{Stratified Train/Test Split:} The dataset was divided into training and testing sets using an 80%–20% split (455 training samples and 114 testing samples). A stratified approach was used so that both sets maintain the same proportion of malignant and benign cases. This helps ensure that model evaluation reflects real-world class distributions.

\item \textbf{Feature Scaling:} All features were standardized using StandardScaler to have a mean of zero and a standard deviation of one. This is important because many machine learning models perform better when features are on a similar scale. To avoid data leakage, the scaler was fitted only on the training data and then applied to the test data.

\item \textbf{Feature Selection:} To reduce complexity and improve efficiency, SelectKBest with the ANOVA F-test (\texttt{f\_classif}) was used to select the 15 most informative features from the original 30. This step helps the model focus on the most relevant characteristics while removing less useful ones.
\end{enumerate}

\subsection{Baseline Models}

To establish a clear starting point, five classical machine learning models were trained using the default settings provided by the scikit-learn library. These models serve as baseline references for evaluating later improvements.

\begin{itemize}
\item \textbf{Logistic Regression (LR):} A simple and widely used linear classifier that estimates probabilities using a logistic function. L2 regularization was applied to prevent overfitting, and the maximum number of iterations was set to 1000 to ensure proper convergence.

\item \textbf{Support Vector Machine (SVM):} This model uses a radial basis function (RBF) kernel to separate data points in a higher-dimensional space. It is effective for handling complex decision boundaries. Probability estimation was enabled to allow evaluation using ROC-AUC.

\item \textbf{K-Nearest Neighbors (KNN):} A non-parametric method that classifies a data point based on the majority class of its $k=5$ nearest neighbors, using Euclidean distance. Its performance depends heavily on properly scaled features.

\item \textbf{Decision Tree (DT):} A tree-based model that splits the data recursively based on feature values, using Gini impurity as the splitting criterion. While easy to interpret, it can overfit the training data if not properly controlled.

\item \textbf{Naive Bayes (NB):} A probabilistic classifier based on Bayes’ theorem, assuming that features are independent. The Gaussian version was used, making it suitable for continuous data.
\end{itemize}

\subsection{Proposed Improvements}

To enhance the performance of the baseline models, four improvement strategies were applied in a structured and systematic manner:

\begin{enumerate}
\item \textbf{Hyperparameter Tuning:} GridSearchCV with 5-fold cross-validation was used to find the best set of parameters for Logistic Regression and SVM. For Logistic Regression, parameters such as regularization strength (C), solver type, and penalty were tuned. For SVM, parameters including C, gamma, and kernel type were optimized. The F1 score was used as the main evaluation metric during this process, as it provides a good balance between precision and recall.

\item \textbf{Random Forest:} A Random Forest model was introduced as an ensemble of multiple decision trees trained on different subsets of the data using bootstrap sampling. Key parameters such as the number of trees (\texttt{n\_estimators}), maximum tree depth (\texttt{max\_depth}), and minimum samples required for splitting (\texttt{min\_samples\_split}) were tuned to improve performance and reduce overfitting.

\item \textbf{Gradient Boosting:} Gradient Boosting was applied as another ensemble technique, where trees are built sequentially, with each new tree focusing on correcting the errors made by previous ones. Important parameters such as the number of estimators, learning rate, and maximum depth were carefully adjusted to achieve better predictive accuracy.

\item \textbf{Soft Voting Ensemble:} Finally, a Soft Voting Ensemble was created by combining the three best-performing models—tuned Logistic Regression, tuned SVM, and Random Forest. Instead of relying on individual predictions, this approach averages the predicted probabilities from each model to make a final decision, resulting in improved overall performance.
\end{enumerate}

\subsection{Evaluation Metrics}

The performance of all models was evaluated using several standard metrics, including Accuracy, Precision, Recall, F1 Score, ROC-AUC, and 5-fold stratified cross-validation F1 score. These metrics provide a comprehensive view of model performance from different perspectives.

In the context of medical diagnosis, Recall (also known as sensitivity) is especially important. This is because false negatives cases where a malignant tumor is incorrectly classified as benign can have serious and potentially life-threatening consequences. Therefore, particular attention was given to Recall when comparing model performance.

\subsection{Noise Robustness Analysis}

To further evaluate the reliability of the models, a noise robustness analysis was conducted. In real-world clinical settings, measurements can vary due to factors such as imaging conditions or human error. To simulate this, Gaussian noise was added to the test data at different levels ($\sigma \in {0.00, 0.05, 0.10, 0.20, 0.30, 0.50}$).

The best-performing model, the tuned SVM, was then tested on these modified datasets, and its Accuracy and F1 Score were recorded. This analysis helps assess how well the model maintains its performance under noisy and imperfect data conditions, which is important for practical clinical use.

Despite this decline at higher noise levels, the model remained relatively stable for low levels of noise ($\sigma \leq 0.10$), which are more representative of real-world clinical measurement variations. \section{Discussion}

\subsection{Why SVM and Logistic Regression Perform Best}

The strong performance of Logistic Regression (F1,=,0.9722) and SVM (F1,=,0.9583) suggests that the dataset becomes largely linearly separable after feature scaling with StandardScaler. Models like Logistic Regression and SVM, which rely on distances and margins between data points, benefit greatly from this normalization.

In contrast, the Decision Tree does not depend on feature scaling and therefore does not gain the same advantage, which helps explain its lower performance. After hyperparameter tuning, the SVM showed further improvement, achieving the highest F1 score of 0.9790 and becoming the best-performing individual model.

\subsection{Impact of Hyperparameter Tuning}

Applying GridSearchCV led to a noticeable improvement in model performance. For SVM, the F1 score increased from 0.9583 to 0.9790, indicating that the default parameter settings are not optimal for this dataset. Careful tuning of parameters such as $C$ and $\gamma$ allowed the model to better capture the underlying patterns in the data.

For Logistic Regression, tuning resulted in a similar Recall value (0.9722) compared to the baseline, but with a slightly improved balance between Precision and Recall.

\subsection{Ensemble Methods Analysis}

The use of ensemble methods further improved model performance. Random Forest helped reduce overfitting compared to a single Decision Tree by combining the predictions of multiple trees trained on different subsets of the data.

The Soft Voting Ensemble achieved the highest ROC-AUC score of 0.9970, outperforming even the tuned SVM. This is because it combines the strengths of multiple models: Logistic Regression, SVM, and Random Forest, leading to better overall probability-based predictions. However, it did not exceed the tuned SVM in terms of F1 score, likely because averaging probabilities can make the model slightly more conservative when making final classification decisions.

\subsection{Clinical Implications of Recall}

In breast cancer diagnosis, false negatives are far more critical than false positives. Misclassifying a malignant tumor as benign can delay treatment and lead to serious consequences.

The tuned SVM achieved a Recall of 0.9722, correctly identifying 41 out of 43 malignant cases (95.35\%) in the test set, with only 2 missed cases. This level of performance is promising for clinical applications. In future work, the decision threshold could be adjusted to further prioritize Recall, even if it results in a slight decrease in Precision, as this trade-off is often acceptable in medical screening scenarios.

\subsection{Feature Insights}

The feature importance results from the Random Forest model show that the “worst” values of tumor measurements play the most important role in classification. This makes intuitive sense from a clinical perspective, as the most extreme characteristics of a tumor are often more informative than average values or variations.

This finding also suggests a practical advantage. It may be possible to build a simpler model using only the top 5–8 most important features while still maintaining strong performance. Such a reduction in required measurements could be especially useful in resource-limited clinical settings, where time and equipment may be constrained.

\subsection{Limitations and Future Work}

Despite the strong results, there are several limitations to consider. First, the dataset contains only 569 samples, which is relatively small. To ensure the model is reliable in real-world applications, it should be tested on larger datasets from multiple clinical sources.

Second, this study focuses only on classical machine learning methods. Future work could include comparisons with deep learning or shallow neural network approaches to evaluate potential performance gains.

Third, while the class imbalance (357 benign vs.\ 212 malignant cases) was handled using stratified splitting, more advanced techniques such as oversampling (e.g., SMOTE) could be explored to further improve Recall.

Finally, improving model interpretability is important for clinical adoption. Techniques such as SHAP values could be used in future work to better explain how the model makes its predictions, helping clinicians trust and understand the results.

\section{Conclusion}

This study presented a detailed comparison of classical machine learning methods for breast cancer diagnosis using the UCI WDBC dataset. Five baseline models were first evaluated, with Logistic Regression showing the strongest initial performance (F1,=,0.9722, AUC,=,0.9914). Building on these results, several improvement strategies like hyperparameter tuning, Random Forest, Gradient Boosting, and a Soft Voting Ensemble were applied to enhance model performance.

Among all models, the tuned SVM achieved the best individual performance, with an F1 score of 0.9790, accuracy of 0.9737, and ROC-AUC of 0.9931. This represents a clear improvement over the baseline SVM. The Voting Ensemble produced the highest ROC-AUC of 0.9970, indicating its strong ability to distinguish between malignant and benign cases. In addition, feature importance analysis highlighted \textit{worst perimeter} as the most influential feature, which aligns with established clinical understanding of tumor characteristics.

Overall, the results show that carefully designed classical machine learning pipelines supported by proper preprocessing, feature selection, and ensemble techniques can achieve high diagnostic accuracy without the need for more complex deep learning models.

For future work, several directions can be explored. These include using techniques such as SMOTE to better handle class imbalance, validating the models on larger and more diverse clinical datasets, and incorporating explainability methods like SHAP to improve transparency and support real-world clinical adoption.
